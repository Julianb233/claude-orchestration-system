<objective>
Voice-friendly shortcut for context window management. Saves current state, determines the next command, then instructs you to `/clear` for a fresh context.

Eliminates context rot by enabling quick refresh cycles while preserving continuity.
</objective>

<process>

<step name="save_state">
**Save Current State**

Create/update `.planning/handoff.md`:

```markdown
# Context Handoff

**Saved:** [ISO timestamp]
**Session:** [brief description from STATE.md current work]

## Current Position
- Phase: [from STATE.md]
- Plan: [current plan if executing one]
- Status: [in-progress | between-tasks | blocked]

## What Was Happening
[1-2 sentence summary of current context]

## Next Command
[The exact GSD command to run after /clear]

## Files Recently Modified
[List from: git status --short]
```

</step>

<step name="determine_next">
**Determine Next Command**

Use the same routing logic as `/gsd:progress`:

1. Check for unexecuted PLAN.md files → `/gsd:execute-plan [path]`
2. Check if phase needs planning → `/gsd:plan-phase [N]`
3. Check if milestone complete → `/gsd:complete-milestone`
4. Check for UAT gaps → `/gsd:plan-phase [N] --gaps`
5. Default → `/gsd:progress`

</step>

<step name="output">
**Output for Easy Copy**

```
============================================
CONTEXT REFRESH READY
============================================

Next command (copy this):

  /gsd:execute-plan .planning/phases/XX-name/XX-NN-PLAN.md

Steps:
1. Run: /clear
2. Paste the command above (or say it to Wispr)

============================================

<gsd-next>/gsd:execute-plan .planning/phases/XX-name/XX-NN-PLAN.md</gsd-next>
```

The `<gsd-next>` tag allows automation scripts to extract the next command.

</step>

</process>

<voice_flow>
## Voice Flow (Wispr)

1. Say: "slash eleven" (or "slash gsd eleven")
2. See: next command displayed
3. Say: "slash clear"
4. Say the next command (or paste it)

</voice_flow>

<integration>
## Integration with gsd.sh Loop

The outer loop script (`scripts/gsd.sh`) parses the `<gsd-next>` tag to automatically run the next command in a fresh Claude instance.

This enables fully autonomous operation:
- Fresh context each iteration (no context rot)
- Memory persists in files (STATE.md, git history)
- Automatic continuation until milestone complete

</integration>

<success_criteria>
- [ ] Handoff file created/updated with current state
- [ ] Next command correctly determined from project state
- [ ] Output clearly shows what to do next
- [ ] `<gsd-next>` tag present for automation
</success_criteria>

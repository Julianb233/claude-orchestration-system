<objective>
Voice-friendly shortcut for rapid situational awareness. Minimal output, maximum clarity.

Shows where you are, what's next, and whether context needs refreshing - in 5 lines.
</objective>

<process>

<step name="quick_read">
**Quick State Read**

Read only what's needed:
- `.planning/STATE.md` - current position line
- Count PLAN.md vs SUMMARY.md in current phase directory

```bash
# Get counts
plans=$(ls -1 .planning/phases/[current-phase]/*-PLAN.md 2>/dev/null | wc -l)
summaries=$(ls -1 .planning/phases/[current-phase]/*-SUMMARY.md 2>/dev/null | wc -l)
```

</step>

<step name="output">
**Output Compact Status**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PHASE [N]: [name]     [██████░░░░] 6/10
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

NEXT: /gsd:execute-plan [path]

CONTEXT: HEALTHY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Progress bar: each █ = 10% complete

</step>

<step name="context_health">
**Context Health Check**

Recommend refresh if ANY of these are true:
- Handoff file suggests long session
- User explicitly asked to check
- Significant work completed this session

Output one of:
- `CONTEXT: HEALTHY` → keep going
- `CONTEXT: ⚠ REFRESH → /11` → recommend refresh

</step>

</process>

<voice_flow>
## Voice Flow (Wispr)

1. Say: "slash twelve" (or "slash gsd twelve")
2. See: phase, progress bar, next command, context health
3. Either:
   - Continue working (context healthy)
   - Say "slash eleven" to refresh (context needs refresh)

</voice_flow>

<comparison>
## Command Comparison

| Command | Purpose | Output |
|---------|---------|--------|
| `/gsd:12` | Quick check | 5 lines |
| `/gsd:progress` | Full report | 30+ lines |
| `/gsd:11` | Context refresh | Prep for /clear |

Use `/12` for "where am I?" checks.
Use `/gsd:progress` for full situational awareness.
Use `/11` when ready to refresh context.

</comparison>

</success_criteria>
